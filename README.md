# MyGame - Mining Simulator

Roblox игра-симулятор добычи брейротов с прогрессивной системой улучшений, наград и монетизацией.

## 📖 Описание проекта

**MyGame** — это интерактивный симулятор добычи брейнротов, где игроки добывают различные типы полезных ископаемых, совершают покупки у торговцев, улучшают свои бомбы и получают различные награды за активность. Игра включает глубокую систему прогрессии с возможностью перерождения (Rebirth) для увеличения множителей добычи.

### Основной геймплей

- 🪨 **Добыча брейнротов** - копайте различные типы брейнротов разного уровня сложности
- 📊 **Система уровней террейна** - каждый брейнрот имеет свой уровень дохода и редкость
- 💰 **Экономическая система** - продавайте добытые предметы, получайте деньги на улучшения
- 🔧 **Улучшения** - покупайте апгрейды для бомбы (диапазон, скорость, вместимость)
- 🎁 **Награды** - ежедневные награды, спины, награды за время игры
- 🔄 **Перерождение** - сбросьте прогресс добычи брейнротов для значительного увеличения множителей

### Система добычи


Каждый уровень добычи (террейн 1-5) имеет свой набор доступных брейнротов:

### Системы улучшений

**Доступные апгрейды:**
- **Range** (+1/+3 диапазон бомбы) - позволяет взрывать и добывать брейнроты на большем расстоянии
- **Carry Capacity** (+1 слот вместимости) - максимум 3 слота
- **Speed** (+1 ускорение ходьбы) - увеличивает скорость перемещения игрока

Каждый апгрейд имеет:
- Базовую стоимость
- Множитель цены (цена растет с каждой покупкой)
- Возможность покупки за Robux

### Система наград

1. **Ежедневные награды** (Daily Rewards)
   - Выдаются один раз в день
   - Растущие награды с каждым днем подряд

2. **Ежедневный спин** (Daily Spin)
   - Механика быстрого получения случайных наград
   - Возможность расширения через Robux

3. **Награды за время игры** (Playtime Rewards)
   - Автоматически начисляются за время, проведенное в игре
   - Различные уровни вознаграждения

### Основные системы

#### 1. **Система загрузки (ServerMain.server.lua)**
- Инициализирует PlayerController первым
- Загружает все контроллеры и модули
- Вызывает методы Init() для инициализации
- Вызывает методы Start() для запуска систем

#### 2. **Контроллеры (Controllers)**

**PlayerController** - управление профилями и данными игроков
- SetupSharedInstances() - создание системных объектов
- Init() - инициализация
- Start() - запуск системы

**BombController** - управление бомбами и добычей
- Отслеживание текущей бомбы
- Система компонентов бомбы

**UpgradesSystem** - система улучшений
- Подсчет стоимости апгрейдов с множителем
- Покупка и применение улучшений
- Ограничения (макс 3 слота для вместимости)

**MonetizationController** - система покупок
- Обработка Game Passes
- Обработка Robux товаров
- Система групповых наград

**LeaderboardController** - система рейтингов
- Отслеживание топовых игроков
- Различные критерии сортировки

#### 3. **Модули (Modules)**

**DataAccessor.lua** - доступ к данным
**ProfileStore.lua** - сохранение профилей
**SellManager.lua** - система продажи предметов
**RebirthSystem.lua** - система перерождения
**BombManager.lua** - система бомб
**LuckyBlockManager.lua** - lucky block комбо
**CarrySystem.lua** - система вместимости инвентаря
**ItemManager.lua** - управление предметами

#### 4. **Конфигурации (ReplicatedStorage/Modules)**

- **BrainRotConfigurations.lua** - характеристики брейнротов, вероятности спавна
- **UpgradesConfigurations.lua** - стоимость и характеристики апгрейдов
- **ProductConfigurations.lua** - ID товаров, Game Passes, награды
- **RarityConfigurations.lua** - система редкости предметов
- **BombsConfigurations.lua** - механика бомб
- **DailyRewardConfiguration.lua** - ежедневные награды
- **DailySpinConfiguration.lua** - конфиг дневного спина

## 🛠️ Используемые технологии

### Основной фреймворк
- **Roblox Studio** - платформа разработки
- **Lua** - язык программирования (Luau диалект с type hints)

### Инструменты разработки
- **Rojo** - инструмент для синхронизации файловой системы с Roblox Studio
  - Конфигурация: `default.project.json`
- **Aftman** - кроссплатформенный менеджер toolchain
  - Конфигурация: `aftman.toml`

### Архитектурные паттерны

1. **MVC (Model-View-Controller)**
   - Controllers - бизнес-логика
   - UI Controllers - управление интерфейсом
   - Modules - бизнес-логика на сервере

2. **Dependency Injection**
   - SharedControllers передается в Init()
   - Позволяет контроллерам обращаться друг к другу

3. **Lifecycle методы**
   - Init() - инициализация
   - Start() - запуск
   - OnClose() - завершение

4. **Event-Driven Architecture**
   - Events папка в ReplicatedStorage
   - Связь между клиентом и сервером

5. **Safe Loading**
   - pcall() для безопасной загрузки модулей
   - Обработка ошибок с логированием
