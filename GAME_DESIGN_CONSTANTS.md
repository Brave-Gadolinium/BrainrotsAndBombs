# Technical Design Document - CONSTANTS
## Руководство геймдизайнера по переменным балансировки

Этот документ описывает все основные константы и конфигурации, необходимые для баланса игры. Все значения находятся в конфигурационных файлах ReplicatedStorage/Modules.

---

## 📋 Constants.lua
**Путь:** `ReplicatedStorage/Modules/Constants.lua`

Основные константы системы игры. Одни из самых критичных переменных для общего баланса.

### Экономика и доход

| Переменная | Описание | Тип | Примечание |
|-------------|---------|-----|-----------|
| `INCOME_SCALING` | Множитель дохода от добычи брейнротов | number | Влияет на общую скорость заработков игроков. Увеличение = больше денег за брейнроты |

### Механика переработки (Recycle System)

| Переменная | Описание | Тип | Примечание |
|-------------|---------|-----|-----------|
| `RECYCLE_MIN_TIME` | Минимальное время переработки в секундах | number | Быстрейшее время, за которое можно переработать брейнрот |
| `RECYCLE_MAX_TIME` | Максимальное время переработки в секундах | number | Самое долгое время переработки для редких брейнротов |

### Механика спавна и жизненный цикл

| Переменная | Описание | Тип | Примечание |
|-------------|---------|-----|-----------|
| `RESPAWN_TIME` | Время появления новых брейнротов в секундах | number | Как часто появляются новые брейнроты для добычи |
| `DROPPED_LIFETIME` | Время жизни выпавшей добычи в секундах | number | Как долго добытые предметы лежат на земле перед исчезновением |

### Системы вместимости и спавна

| Переменная | Описание | Тип | Примечание |
|-------------|---------|-----|-----------|
| `MAX_ITEMS_PER_MINE` | Максимум брейнротов в одной шахте одновременно | number | Предотвращает лаги от избытка объектов |
| `MIN_ITEM_SPACING` | Минимальное расстояние между спавнящимися брейнротами | number | Помогает избежать наложения объектов при спавне |

### Система уровней и редкости

| Переменная | Описание | Тип | Примечание |
|-------------|---------|-----|-----------|
| `SPAWNER_TIERS` | Конфигурация уровней шахт | table | Определяет, какие брейнроты появляются на каких уровнях |
| `DEFAULT_CHANCE` | Базовый шанс для системы редкости | percent | Используется в формулах вероятности появления |

### Система мутаций

| Переменная | Описание | Тип | Примечание |
|-------------|---------|-----|-----------|
| `MUTATIONS` | Список доступных мутаций предметов | table | Определяет какие модификации могут быть у брейнротов |
| `MUTATION_MULTIPLIERS` | Множители дохода для каждой мутации | table | На сколько % увеличивается цена при наличии мутации |

### Пример использования в коде:
```lua
local Constants = require(ReplicatedStorage.Modules.Constants)

-- Получение константы
local incomeMultiplier = Constants.INCOME_SCALING
local respawnTime = Constants.RESPAWN_TIME

-- Использование в логике
local totalIncome = baseIncome * Constants.INCOME_SCALING
task.wait(Constants.RESPAWN_TIME)
```

## 💣 BombsConfigurations.lua
**Путь:** `ReplicatedStorage/Modules/BombsConfigurations.lua`

Конфигурация всех доступных бомб в игре. Каждая бомба - это инструмент для добычи брейнротов с уникальными характеристиками.

### Структура бомбы

```lua
{
  Id = "bomb_id",              -- Уникальный идентификатор
  DisplayName = "Бомба 1",     -- Имя в UI
  Cooldown = 2,                -- Cooldown в секундах между взрывами
  ExplosionRadius = 25,        -- Радиус взрыва в studs
  Price = 0,                   -- Стоимость покупки в игровой валюте
  RobuxPrice = 0,              -- Стоимость в Robux (0 = не продается за Robux)
  ImageId = "rbxassetid://...", -- ID изображения для UI
  Damage = 10,                 -- Урон по брейнротам
  Level = 1                    -- Уровень/телепорт бомбы
}
```

### Параметры баланса для каждой бомбы

| Параметр | Описание | Влияние на геймплей |
|----------|---------|-------------------|
| **DisplayName** | Видимое имя в интерфейсе | Влияет только на UI |
| **Cooldown** | Время ожидания между взрывами | Меньше cooldown = быстрее добывать, но энергозатратнее |
| **ExplosionRadius** | Площадь повреждения взрыва | Больше радиус = больше брейнротов за один взрыв |
| **Price** | Базовая стоимость в монетах | Ранние бомбы дешевле, поздние дороже |
| **RobuxPrice** | Цена в премиум валюте | 0 = недоступна за Robux |
| **Damage** | Урон по брейнротам | Выше = быстрее разрушает брейнроты |
| **Level** | Уровень доступности | Разблокируется в зависимости от прогресса |

### Рекомендации по баланс-дизайну

**Для новичков (Level 1-2):**
- Низкий cooldown (2-3 сек)
- Маленький радиус (20-25 studs)
- Низкий урон (5-10)
- Доступная цена (0-500 монет)

**Для среднего контента (Level 3-5):**
- Средний cooldown (1.5-2.5 сек)
- Средний радиус (30-40 studs)
- Средний урон (15-25)
- Средняя цена (1000-10000 монет)

**Для эндгейма (Level 6+):**
- Низкий cooldown (0.5-1.5 сек)
- Большой радиус (45-60 studs)
- Высокий урон (30-50)
- Высокая цена (50000+ монет)

## 🪨 RarityConfigurations.lua
**Путь:** `ReplicatedStorage/Modules/RarityConfigurations.lua`

Система редкости брейнротов. Определяет шансы появления и множители цены для каждого уровня редкости.

### Уровни редкости

| Редкость | Вероятность | Множитель цены | Цвет | Описание |
|----------|-------------|----------------|------|----------|
| **Common** | 60% | 1.0x | Серовато | Брейнроты часто встречаются |
| **Uncommon** | 25% | 1.5x | Синий | Редче, чем обычные |
| **Rare** | 10% | 2.5x | Фиолетовый | Редкие брейнроты |
| **Epic** | 4% | 5.0x | Оранжевый | Очень редкие |
| **Legendary** | 1% | 10.0x | Золотой | Легендарные брейнроты |

### Параметры баланса

```lua
RarityConfigurations.Rarities = {
  ["Common"] = {
    Weight = 600,              -- Вес в системе вероятности (выше = чаще)
    PriceMultiplier = 1.0,     -- На сколько умножается базовая цена
    ExperienceMultiplier = 1.0 -- Бонус опыта (если есть)
  },
  -- остальные редкости...
}
```

### Советы по настройке

- **Увеличить редкость легендарных предметов:** уменьшите вес Common/Uncommon
- **Снизить вознаграждение за редкие:** уменьшите PriceMultiplier
- **Более прогрессивное развитие:** сделайте разницу множителей больше (1x → 15x вместо 1x → 10x)

## 🧬 BrainRotConfigurations.lua (ItemConfigurations.lua)
**Путь:** `ReplicatedStorage/Modules/ItemConfigurations.lua`

Полная конфигурация каждого типа брейнрота в игре. Это основные предметы, которые добывают игроки.

### Структура брейнрота

```lua
{
  Name = "Stone",              -- Уникальное имя (код)
  DisplayName = "Камень",      -- Видимое имя в UI
  Rarity = "Common",           -- Уровень редкости (Common/Uncommon/Rare/Epic/Legendary)
  BaseIncome = 10,             -- Базовая цена при продаже монет
  ImageId = "rbxassetid://...", -- ID изображения в инвентаре
  Description = "...",         -- Описание предмета
  Health = 2,                  -- Прочность брейнрота (сколько ударов нужно)
  SpawnerTiers = {1},          -- В каких уровнях (шахтах) появляется
  Weight = 100                 -- Вес в системе вероятности спавна
}
```

### Типы брейнротов и их параметры

| Название | DisplayName | Редкость | BaseIncome | Health | Уровни | Примечание |
|----------|-------------|----------|-----------|--------|--------|-----------|
| Stone | Камень | Common | 10 | 2 | 1 | Самый базовый брейнрот |
| Sapphire | Сапфир | Uncommon | 50 | 10 | 2-3 | Голубой камень |
| Emerald | Изумруд | Rare | 150 | 25 | 3-4 | Зелёный камень |
| Sunstone | Солнечный камень | Rare | 300 | 50 | 4-5 | Жёлтый камень |
| Rhodonite | Родонит | Epic | 750 | 100 | 5 | Редкий красный камень |

### Параметры баланса для брейнротов

| Параметр | Описание | Как это влияет |
|----------|---------|----------------|
| **BaseIncome** | Базовая цена при продаже | Главный источник денег. Увеличение = быстрее накопить на апгрейды |
| **Health** | Сколько ударов нужно для добычи | Выше = дольше добывать, но больше опыта/денег |
| **Weight** | Вероятность появления | Выше = чаще встречается этот брейнрот |
| **SpawnerTiers** | На каких уровнях появляется | Определяет прогрессию контента |

### Формула итоговой цены брейнрота:

```
Итоговая цена = BaseIncome × RarityMultiplier × MutationMultiplier × RebirthMultiplier
```

### Рекомендации по балансу

**Для новичков (Stone):**
- BaseIncome: 5-15
- Health: 2
- Weight: 600 (очень часто)

**Для среднего уровня (Sapphire/Emerald):**
- BaseIncome: 50-200
- Health: 10-25
- Weight: 250-350

**Для эндгейма (Sunstone/Rhodonite):**
- BaseIncome: 300-1000
- Health: 50-100
- Weight: 50-100

## 🎯 UpgradesConfigurations.lua
**Путь:** `ReplicatedStorage/Modules/UpgradesConfigurations.lua`

Конфигурация всех доступных апгрейдов (улучшений), которые можно купить за монеты.

### Структура апгрейда

```lua
{
  Id = "Range1",                    -- Уникальный ID
  StatId = "BonusRange",            -- Параметр игрока, который увеличивается
  Amount = 1,                       -- На сколько увеличивается каждый раз
  DisplayName = "+1 диапазон",     -- Видимое имя
  BaseCost = 250,                   -- Базовая стоимость первого уровня
  PriceMultiplier = 1.5,            -- На сколько увеличивается цена с каждым уровнем
  RobuxProductId = 123456,          -- ID Robux товара (0 = только за монеты)
  ImageId = "rbxassetid://..."     -- Иконка в магазине
}
```

### Доступные апгрейды

| ID | Статистика | Эффект | BaseCost | Множитель | Max Level |
|-----|-----------|--------|----------|-----------|-----------|
| **Range1** | BonusRange | +1 диапазон | 250 | 1.5x | ∞ |
| **Range3** | BonusRange | +3 диапазона | 250 | 1.5x | ∞ |
| **Carry1** | CarryCapacity | +1 слот | 50000 | 2.5x | 3 |
| **Speed1** | BonusSpeed | +1 скорость | 500 | 1.5x | ∞ |

### Формула стоимости апгрейда:

```
Цена на уровне N = BaseCost × (PriceMultiplier ^ N)
```

**Пример:**
- BaseCost = 250
- Множитель = 1.5
- Уровень 1: 250 × 1.5^0 = 250
- Уровень 2: 250 × 1.5^1 = 375
- Уровень 3: 250 × 1.5^2 = 562
- Уровень 10: 250 × 1.5^9 = 28,125

### Рекомендации по баланс-дизайну

**Для быстрого прогресса (низкий множитель 1.3-1.5):**
```lua
BaseCost = 100,
PriceMultiplier = 1.3
```
Игрок может часто покупать, быстро прокачиваться.

**Для среднего прогресса (средний множитель 2.0-2.5):**
```lua
BaseCost = 1000,
PriceMultiplier = 2.0
```
Нужно времени на накопление, но доступно регулярно.

**Для эндгейма (высокий множитель 3.0+):**
```lua
BaseCost = 100000,
PriceMultiplier = 3.0
```
Требует значительных денег, прогресс замедляется.

## 📊 Чек-лист геймдизайнера

При изменении баланса проверьте:

- [ ] **Прогрессия новичков** - первые 30 минут не скучно и не сложно
- [ ] **Кривая сложности** - каждый новый уровень добычи сложнее, но доступнее
- [ ] **Экономический баланс** - цены апгрейдов не слишком высокие, но требуют затрат
- [ ] **Редкость предметов** - легендарные брейнроты встречаются редко (1-5%)
- [ ] **Монетизация** - Game Passes и Robux товары полезны, но не обязательны
- [ ] **Социальные системы** - ежедневные награды мотивируют возвращаться
- [ ] **Эндгейм контент** - система перерождения дает достаточную мотивацию для повторного прохождения

## 🔧 Быстрые советы

### Если игра становится слишком лёгкой:
1. Уменьшите `INCOME_SCALING` в Constants
2. Увеличьте `BaseCost` в UpgradesConfigurations
3. Уменьшите `Weight` редких брейнротов
4. Увеличьте `Health` (Прочность) брейнротов

### Если игра становится слишком сложной:
1. Увеличьте `INCOME_SCALING`
2. Уменьшите `BaseCost` апгрейдов
3. Уменьшите `PriceMultiplier` апгрейдов
4. Увеличьте вероятность редких предметов (Weight)

### Для лучшей монетизации:
1. Сделайте Game Passes полезными (+50% доход минимум)
2. Используйте Robux товары для ускорения, а не для P2W
3. Ежедневные награды должны быть привлекательными но не чрезмерными
